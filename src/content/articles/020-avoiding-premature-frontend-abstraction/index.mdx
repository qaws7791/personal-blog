## 성급한 추상화와 코드

많은 코드가 특정 페이지나 컴포넌트에 종속성을 가짐.
이러한 코드를 공통 폴더에 전부 집어넣으면 많은 문제가 발생할 수 있음.
실제로 재사용이 불가능한 코드들이 공통 폴더에 배치되고, 실제로 원하는 코드를 찾을 때 방해가 될 수 있음.
또한 종속성이 프로젝트 전체에 흩어지게 되고, 유지보수하기 어려워짐

## Nextjs Private Folder

Nextjs는 페이지 컴포넌트(`page.tsx`) 옆에 다른 코드를 배치할 수 있도록 비공개 폴더를 제공하고 있음. 비공개 폴더는 폴더 이름 앞에 `_`를 접두사를 통해 만들 수 있습니다. 이러한 `_components`, `_hooks` 폴더의 코드는 해당 페이지에서만 사용됩니다. 추후 이러한 코드들이 재사용이 필요하다면 공통 폴더로 배치시키고 사용하면 됩니다.

```tsx
app/
├── dashboard/
│   ├── _components/
│   │   ├── DashboardHeader.tsx
│   │   ├── StatsCard.tsx
│   │   └── QuickActions.tsx
│   ├── _hooks/
│   │   └── useDashboard.ts
│   ├── analytics/               # 하위 라우트
│   │   ├── _components/
│   │   │   ├── AnalyticsChart.tsx
│   │   │   └── MetricCard.tsx
│   │   ├── _hooks/
│   │   │   └── useAnalytics.ts
│   │   └── page.tsx
│   ├── settings/               # 하위 라우트
│   │   ├── _components/
│   │   │   ├── SettingsForm.tsx
│   │   │   └── SettingsToggle.tsx
│   │   ├── _hooks/
│   │   │   └── useSettings.ts
│   │   └── page.tsx
│   └── page.tsx
```
